# v0.5 硬件审查

依据 `硬件/v0.5/easyeda_schematic_final.md` 和 BOM 整理。

## 核心器件

| 功能 | 器件 |
| --- | --- |
| MCU | GD32F303CCT6，256 KiB Flash，48 KiB SRAM |
| 无线 | BOM 为 E28-2G4M20SX；现有 M20S 手册标明核心为 SX1281 |
| USB | USB-C，USB FS Device，外部 1.5 kOhm D+ 上拉 |
| 主电源 | SY8089AAAC 降压至 3V3_REG |
| 电源选择 | TPS2116DRLR，在 3V3_REG 与 TGT_3V3 间选择 |
| 目标供电 | 两路 SY6280AAAC，输出 5 V 和 3.3 V |

## 投板前必须复核

1. `LED1` 的 K 接 `3V3_MAIN`、A 经 R5 接 GND。按常规定义该 LED 会反向偏置，
   无法正常点亮。请在 EasyEDA 中核对符号引脚定义、封装极性和实物方向。
2. RGB LED 使用 PC13/PC14/PC15 低电平灌电流。R18/R26 为 470 Ohm，理论电流可能
   超过这些低驱动能力引脚的推荐值。必须对照 GD32F303 数据手册的 PC13-PC15
   输出能力确认，必要时增大电阻或加驱动管。
3. USB 需要稳定的 48 MHz 时钟。8 MHz HXTAL 到 120 MHz 系统时钟的配置必须同时
   给 USB 提供准确 48 MHz，不能仅验证 CPU 能运行。
4. `TGT_3V3` 既可作为本板 TPS2116 输入，又可由本板 SY6280 输出。固件必须避免在
   外部目标已供电时打开 `TGT3V3_EN`，否则存在电源互灌风险。仅靠软件仍不如硬件
   防反灌可靠。
5. `D3` 从 `TGT_5V` 指向 `5V_SYS`，会允许目标 5 V 给本板供电。确认这是预期行为，
   并核算肖特基压降、USB 反灌路径及 TPS2116 工作状态。
6. `MCU_TGT_NRST` 和 `MCU_TGT_BOOT` 串联 1 kOhm。需要结合目标端输入电容检查复位
   边沿；若要兼容高速 SWJ 序列，不应把这两个网络当普通慢速 GPIO 随意处理。
7. E28 模块的 TX_EN/RX_EN 时序、BUSY 极性、SPI 最高频率和复位时间必须以模块官方
   手册为准。

## 板级安全态

- `TGT5V_EN = 0`
- `TGT3V3_EN = 0`
- `RF_TX_EN = 0`
- `RF_RX_EN = 0`
- `RF_NSS = 1`
- `RF_NRESET = 0`，初始化完成后再释放
- `USB_DP_PU = 0`，USB 协议栈就绪后再连接
- 目标 `NRST` 与 `BOOT` 默认高阻输入

## 首次上电

1. 不连接目标板，不焊或不启用射频模块，限流电源设为 5 V/100 mA。
2. 测量 `5V_SYS`、`3V3_REG`、`3V3_MAIN`、`VDDA`，检查纹波和异常发热。
3. 通过 H1 下载最小固件，确认 8 MHz HXTAL 和 120 MHz 系统时钟。
4. 只测试 RGB LED 与按键，不打开目标供电。
5. 检查 USB D+/D- 静态阻抗，之后再启用软件上拉和 USB 枚举。
6. 单独验证两路目标电源的空载、限流和短路保护。
7. 最后焊接或启用射频模块，先读状态寄存器，再进行发射测试。
